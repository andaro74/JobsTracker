## Jobs Tracker Spring WebFlux Project

Demonstrate a Jobs Tracker REST application via Spring WebFlux, DynamoDB and Kafka.

### ID Conventions

This API uses **string business identifiers** for most domain objects instead of raw UUIDs:

- `customerId` (String): business identifier for a customer. Example: `CUST-0001` or a ULID-like string generated by the system.
- `catalogId` (String): business identifier for a catalog item. Example: `CAT-0001` or a ULID-like string.
- `contractorId` (String): business identifier for a contractor. Example: `CON-0001` or a ULID-like string.
- `jobId` (String): business identifier for a job item. DynamoDB PKs are derived from these IDs (e.g. `JobNumber#<jobId>`).

The underlying DynamoDB tables store composite keys using these IDs with prefixes (e.g. `CustomerNumber#<customerId>`, `CatalogNumber#<catalogId>`, `ContractorNumber#<contractorId>`), but the external REST API works exclusively with the bare string IDs.

### REST APIs

#### Customers

- `POST /api/customers` – create a customer. The server generates a `customerId` (String).
- `GET /api/customers/{customerId}` – retrieve a customer by its string `customerId`.
- `PUT /api/customers/{customerId}` – update a customer identified by `customerId`.
- `DELETE /api/customers/{customerId}` – delete a customer by `customerId`.

Example URL:

```http
GET /api/customers/CUST-0001
```

#### Catalog

- `POST /api/catalog` – create a catalog item. The server generates a `catalogId` (String).
- `GET /api/catalog/{catalogId}` – retrieve a catalog item by its string `catalogId`.
- `PUT /api/catalog/{catalogId}` – update a catalog item identified by `catalogId`.
- `DELETE /api/catalog/{catalogId}` – delete a catalog item by `catalogId`.

Example URL:

```http
GET /api/catalog/CAT-0001
```

#### Contractors

- `POST /api/contractors` – create a contractor. The server generates a `contractorId` (String).
- `GET /api/contractors/{contractorId}` – retrieve a contractor by its string `contractorId`.
- `DELETE /api/contractors/{contractorId}` – delete a contractor by `contractorId`.
- `GET /api/contractors?zipCode=XXXXX` – list contractors filtered by ZIP code.

Example URL:

```http
GET /api/contractors/CON-0001
```

#### Jobs

Jobs reference `catalogId` and `customerId` as strings in their DTOs/requests:

- `JobItemDTO.catalogId` – string catalog identifier for the job.
- `JobItemDTO.customerId` – string customer identifier for the job.
- `CreateJobItemDTO.catalogId` / `CreateJobItemDTO.customerId` – same string identifiers in create/update payloads.
